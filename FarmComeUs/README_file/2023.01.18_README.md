### [DateBase]데이터 무결성(Data Inteqrity)을 위한 DB 수정


한명의 member는 자신의 store를 만들 수도 있고 만들지 않을 수도 있다.
한 member의 스토어 여부를 나타내기 위해 member 테이블에 exist\_store 컬럼을 넣어주었다.
하지만 이렇게 되면 데이터 무결성이 깨질 수 있다는 컨설턴트님의 피드백..!

만약 exist\_store 컬럼 값이 True인데 store가 없다면 데이터 관계 무결성과 관련해서 문제가 생길 여지가 있다. 
store가 만들어지면 member\_id 값이 store 테이블에 들어가니, 유저의 스토어 여부를 확인하고 싶다면 store 테이블을 join하는 방식으로 수정
